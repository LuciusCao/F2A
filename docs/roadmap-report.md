## 1. 执行摘要

F2A 是一个基于 libp2p 的去中心化 Agent 协作网络，已实现 P2P 通信、能力发现、任务委托、信誉系统等核心功能。代码质量较高，但存在以下关键问题需要优先解决：

### 核心发现

| 维度 | 评分 | 关键问题 |
|------|------|---------|
| 架构设计 | 7.3/10 | 节点身份持久化缺失 |
| 协作机制 | 7.0/10 | 能力量化不足，无比较优势调度 |
| 部署支持 | 6.5/10 | mDNS 未实现，DHT 默认禁用 |
| 用户体验 | 5.5/10 | 安装配置复杂 (7 步/15+ 配置项) |
| 安全性 | 8.5/10 | 3 个 P0 漏洞，5 个 P1 风险 |

### 优先级行动项

1. **P0 (本周)**: 节点身份持久化、CORS 配置验证、引导节点验证
2. **P1 (1-2 周)**: mDNS 实现、配置简化、安全漏洞修复
3. **P2 (1 个月)**: 能力量化模型、比较优势调度、可视化面板

## 2. 产品定位

### 2.1 核心价值主张

> **让每个 OpenClaw Agent 成为网络节点，实现去中心化的能力发现、任务委托与协作执行。**

### 2.2 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 个人开发者 | 拥有多台设备 (MacBook、树莓派等) | 设备间任务协同、负载分担 |
| 小型团队 | 3-10 人，多 Agent 部署 | 团队内 Agent 能力共享、任务分发 |
| 技术爱好者 | 对 P2P、去中心化感兴趣 | 体验分布式协作、搭建私有网络 |
| 企业用户 (远期) | 多部门、多地域部署 | 安全可控的内部 Agent 网络 |

### 2.3 差异化优势

| 维度 | F2A | 中心化协作平台 |
|------|-----|---------------|
| 架构 | P2P 去中心化 | 中心服务器 |
| 隐私性 | 高 (本地网络优先) | 低 (数据经过中心) |
| 配置复杂度 | 中 (待简化) | 低 |
| 安全模型 | 信誉系统 + 白名单 | 账号体系 |

## 5. 安全问题清单

### 5.1 P0 级漏洞 (立即修复)

| 编号 | 问题 | 风险 | 状态 | 预计完成 |
|------|------|------|------|---------|
| P0-1 | 生产环境 CORS 配置过于宽松 | CSRF、数据泄露 | ⏳ 待修复 | 2026-03-14 |
| P0-2 | 私钥存储无加密保护 | 历史通信可解密 | ⏳ 待修复 | 2026-03-19 |
| P0-3 | 引导节点信任链未验证 | 中间人攻击 | ⏳ 待修复 | 2026-03-14 |

### 5.2 P1 级风险 (本周修复)

| 编号 | 问题 | 风险 | 状态 | 预计完成 |
|------|------|------|------|---------|
| P1-1 | 速率限制器内存泄漏 | 内存耗尽 DoS | ⏳ 待修复 | 2026-03-15 |
| P1-2 | 挑战机制缺少经济约束 | 恶意挑战攻击 | ⏳ 待修复 | 2026-03-15 |
| P1-3 | 日志可能泄露敏感信息 | 信息收集攻击 | ⏳ 待修复 | 2026-03-17 |
| P1-4 | DHT 路由表无大小限制 | 路由表污染 | ⏳ 待修复 | 2026-03-15 |
| P1-5 | Webhook 无显式 TLS 验证 | 中间人攻击 | ⏳ 待修复 | 2026-03-15 |

### 5.3 安全基线配置

```bash
# .env.production
NODE_ENV=production
F2A_CONTROL_TOKEN=$(openssl rand -hex 32)
F2A_SIGNATURE_KEY=$(openssl rand -hex 32)
F2A_ALLOWED_ORIGINS=https://your-domain.com
F2A_SECURITY_LEVEL=high
F2A_RATE_LIMIT_MAX_REQUESTS=30
F2A_LOG_LEVEL=WARN
F2A_CHALLENGE_MIN_STAKE=30
```

## 9. 核心决策摘要

### 9.1 架构决策

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 身份存储 | 明文/加密/HSM | 加密存储 | 平衡安全与易用性 |
| 发现机制 | mDNS/DHT/混合 | 混合模式 | 局域网 mDNS + 广域网 DHT |
| 任务分配 | 随机/比较优势 | 比较优势 | 优化网络效率 |
| 信誉系统 | 单一维度/多维度 | 多维度 | 更精细的信任评估 |

### 9.2 产品决策

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| 目标用户 | 个人/企业/混合 | 个人优先 | 降低入门门槛，快速验证 |
| 部署方式 | SaaS/自托管/混合 | 自托管优先 | 隐私保护，符合 P2P 理念 |
| 商业化 | 免费/增值/企业版 | 开源 + 增值 | 积累用户后变现 |

### 9.3 安全决策

| 决策 | 选项 | 选择 | 理由 |
|------|------|------|------|
| CORS 配置 | 宽松/严格 | 生产环境严格 | 防止 CSRF 攻击 |
| 引导节点 | 信任/验证 | 公钥指纹验证 | 防止中间人攻击 |
| 私钥存储 | 明文/加密 | 加密存储 | 防止服务器入侵 |
